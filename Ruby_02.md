## 1. 기본구조 파악하기
### 서버가 하는 일
Rails 서버는 MVC 패턴으로 이루어져 있기 때문에 Model, View, Controller가 서로 상호작용하여 정보를 가공하게 된다.

-   Model : 어플리케이션의 데이터와 정보를 다루는 규칙을 담당한다
-   View : 데이터 표현에 대한 부분을 담당한다
-   Controller: Model과 View를 이어주며, 데이터 가공을 수행한다.
